# TOOL-DRAWERS — the 61 SCL Glyphs as Graph Parti Tool Organization

> The 61 Diameter glyphs are dump drawers. Each glyph is a bucket that holds
> related tools. Right-click a glyph on the bottom row = pop out the drawer.
> Left-click = activate the last-used tool from that drawer. Alt+type = command search.

> The mapping is polysemic — the glyph means what it means in the SCL system,
> AND it means that drawer of tools. The mirror glyph holds the mirror tool.
> The builder holds drawing tools. The spectator holds inspection tools.
> Some are on-the-nose obvious. Some have unexpected homes. All have room to grow.

## Implementation Notes

**Right-click glyph → drawer popup:**
Each glyph on the bottom SCL band gets a right-click context menu listing its tools.
Clicking a tool in the menu activates it. The glyph remembers the last-used tool.

**Left-click glyph → activate last-used:**
If the glyph has a tool drawer, left-click activates the most recently used tool from
that drawer (instead of typing the glyph character into text).

**Alt → command mode:**
Tap Alt to open a floating command input. Type a tool name (fuzzy-matched). Enter activates.
Escape cancels. Same input can later accept NLP queries when the parser is wired.

**Future: NLP layer:**
The command input becomes the front door for natural language: "I need to round this corner"
→ fuzzy matches to fillet. "repeat this in a circle" → polar array. The keyword hunt is
the bridge between typing tool names and describing what you need.

**Future: procedural scripting:**
The same input accepts Python-like commands that generate geometry procedurally.
`rect(0, 0, 5, 3)` → draws a rectangle. The scripting layer is the text-to-geometry pipe.
